# Update Documentation Workflow

## Overview

This repository contains a GitHub Actions workflow that automatically regenerates the runbook documentation in the `docs/` folder based on the PowerShell runbooks and the documentation/metadata generated by the helper scripts in `.github/scripts/`.

Workflow file: `.github/workflows/update-documentation.yml`

## When it runs

- **Manual**: via `workflow_dispatch` (Run workflow button in GitHub Actions)
- **On push to `master`** when changes include:
  - `**.ps1`
  - `device/**`, `group/**`, `org/**`, `user/**`
  - `.permissions/**`

## What it does (high level)

1. **Cleans generated docs output**
   - Removes everything under `docs/` **except** `docs/general/` and `docs/README.md`.
2. **Generates detailed runbook docs** (separate folder per runbook)
   - Uses `.github/scripts/Get-RunbookDescriptions.ps1` (includes permissions, parameters, notes, links, etc.).
3. **Generates a single-file overview**
   - Writes `docs/other/RealmJoin-RunbookOverview.md`.
4. **Generates list-based documentation**
   - Overview lists, compact lists, permission list, parameter list, custom runbook list, automation account name list.
5. **Generates JSON outputs**
   - Runbook detail JSON and a unique permissions collection JSON.
6. **Commits and pushes** changes back to the repository
   - Only if there are staged changes (`git diff --staged --quiet` is used to detect changes).

## Output locations

- `docs/` (detailed docs per runbook and TOCs)
- `docs/lists/` (generated list documents)
- `docs/other/` (overview + reports)
- `docs/other/json/` (JSON exports)

## How to run it manually

### Option A: Run the workflow in GitHub

1. Go to **Actions** → **📚 Update documentation**
2. Click **Run workflow** (select branch `master` if needed)

### Option B: Run the generation scripts locally (for preview)

You can run the underlying scripts locally with PowerShell to preview generated output before pushing changes.

Typical steps:

1. From the repo root, start PowerShell: `pwsh`
2. Run the same scripts as the workflow uses (see `.github/workflows/update-documentation.yml`)

Note: The workflow runs on **Ubuntu** (`ubuntu-latest`). If you run locally on macOS/Windows, make sure you use PowerShell 7 (`pwsh`) and that path handling matches your environment.

## Notes

- The workflow has `contents: write` permission because it commits documentation changes.
- `docs/general/` is treated as “handwritten” documentation and is kept during the cleanup step.
